# 메인화면 기획서

> 최종 업데이트: 2026-04-13

## 1. 화면 구조

```
+----------------------------------------------------------+
| TopToolbar                                               |
+------------+---------------------------------------------+
| LeftSidebar| MainCalendar                                |
| (접이식)    |                              [RightPanel]  |
|            |                              (오버레이)      |
+------------+---------------------------------------------+
```

- **TopToolbar**: 상단 고정, 네비게이션 및 앱 컨트롤
- **LeftSidebar**: 접이식 사이드바 (토글 가능, 상태 유지)
- **MainCalendar**: 월간 캘린더 그리드
- **RightPanel**: 선택한 날짜의 이벤트 목록 (오버레이, 캘린더 위에 덮음)

---

## 2. TopToolbar

### 2.1 좌측 영역 (사이드바 너비와 동기화)
- **햄버거 메뉴**: 사이드바 토글 (열기/닫기)
- **로고 + 서비스명**: 사이드바 열린 상태에서만 노출 ("To-do Calendar")

### 2.2 중앙 영역
- **월 네비게이션**: `< 2026년 4월 >` 형태, 좌우 화살표로 월 이동
- **오늘 버튼**: 클릭 시 오늘 날짜로 이동 (중앙캘린더 + 사이드바캘린더 + 선택날짜 모두 리셋)

### 2.3 우측 영역
- **Archive 버튼**: 완료된 할일 페이지(`/done`)로 이동
- **Settings 버튼**: 설정 페이지(`/settings`)로 이동

---

## 3. LeftSidebar (좌측 패널)

### 3.1 접이식 동작
- 햄버거 메뉴로 토글
- 열림/닫힘 상태는 `localStorage`에 저장하여 새로고침 시 유지
- 닫힌 상태: 너비 0 (완전히 숨김)

### 3.2 이벤트 추가 버튼
- 미니캘린더 상단에 위치
- 캡슐(pill) 형태 버튼: `+ Create ▼`
- 클릭 시 타입 선택 팝업 표시 (Todo / Schedule)
  - Todo 선택 → `/todos/new` 모달 오픈
  - Schedule 선택 → `/schedules/new` 모달 오픈

### 3.3 미니캘린더
- shadcn/ui Calendar 기반
- **독립적 월 관리**: 중앙캘린더와 별도의 월 상태 (`sidebarMonth`)
- 월 이동 규칙:
  - 좌측캘린더 달 변경 → 중앙캘린더 영향 없음
  - 중앙캘린더 달 변경 → 좌측캘린더 영향 없음
  - 좌측캘린더 날짜 선택 → 중앙캘린더 해당 월로 싱크
  - "오늘" 버튼 → 양쪽 모두 오늘 달로 리셋
- 날짜 선택:
  - 클릭 → 해당 날짜 선택 + 중앙캘린더 월 싱크 + 우측패널 오픈
  - 이미 선택된 날짜 재클릭 → 선택 해제 + 우측패널 닫기
- 초기 상태: 오늘 날짜가 선택됨
- 공휴일/일요일은 빨간색 텍스트

### 3.4 이벤트 종류 (태그 목록)
- 헤더: "이벤트 종류" + `>` 태그관리 진입 아이콘
- 기본 제공 태그 (항상 최상단):
  1. **기본** — 기본 태그 색상
  2. **공휴일** — 공휴일 태그 색상
- 이후 사용자가 생성한 태그들
- 각 태그 행:
  - 원형 체크마크 아이콘 (활성=색상 채움, 비활성=테두리만)
  - 클릭으로 토글 → 해당 태그의 이벤트를 캘린더/패널에서 숨기거나 표시
  - 숨김 상태는 `localStorage`에 저장
- `>` 아이콘 클릭 → 태그 관리 페이지(`/tags`) 모달 오픈

## 4. MainCalendar (중앙 캘린더)

### 4.1 그리드 구조
- 7열 (일~토) × N행 (해당 월의 주 수)
- 요일 헤더 행: 일요일은 빨간색
- 각 행은 균등 높이로 분배 (flex-1)

### 4.2 날짜 셀
- 날짜 숫자 표시 (상단 좌측)
- 셀 전체 배경으로 상태 표시:
  - **선택된 날**: 진한 배경 + 흰색 텍스트
  - **오늘 (미선택)**: 연한 배경
  - **이전/다음 달**: 흐린 텍스트
  - **공휴일/일요일**: 빨간색 텍스트
- 공휴일 셀: 마우스 호버 시 공휴일 이름 tooltip

### 4.3 이벤트 바 표시 (Desktop)
- 날짜 숫자 아래 영역에 이벤트 바 표시
- **이벤트 스태킹 알고리즘**: Greedy bin-packing
  - 긴 이벤트 우선 배치 → 남은 공간에 짧은 이벤트 채움
  - 다일 이벤트는 여러 열에 걸쳐 하나의 바로 스팬
- 표시 가능 행 수: 셀 실제 높이에서 동적 계산 (ResizeObserver)
- 초과 이벤트: `+N more` 표시
- **이벤트 타입별 렌더링**:
  - `at` (시점): 투명 배경 + 좌측 색상 dot + 진한 텍스트
  - `period`/`allday` (기간): 색상 배경 바 + 좌측 흰색 dot + 흰색 텍스트
- 이벤트 바 클릭 → EventPreviewCard 팝업 표시

### 4.4 이벤트 바 표시 (Mobile)
- 이벤트 바 대신 최대 3개 색상 dot으로 요약 표시

### 4.5 셀 클릭 동작
- 날짜 셀 클릭 → 해당 날짜 선택 + 우측패널 오픈
- 같은 날짜 재클릭 → 선택 해제 + 우측패널 닫기
- 이벤트 바 클릭은 셀 클릭과 별개 (stopPropagation)

### 4.6 태그 필터링
- 숨긴 태그의 이벤트는 캘린더에 표시되지 않음

## 5. RightPanel (우측 이벤트 패널)

### 5.1 패널 동작
- 오버레이 방식: 중앙 캘린더 위에 덮어서 표시
- 슬라이드 인/아웃 애니메이션
- 닫기 버튼: 우측 상단 `>` 화살표

### 5.2 패널 구성

**스크롤 영역** (상단~중간):

#### 5.2.1 날짜 헤더
- 선택된 날짜: "2026년 4월 13일" (대형 텍스트)
- 요일: "일요일" (소형 텍스트)
- 날짜 미선택 시 비표시

#### 5.2.2 Foremost Event 배너 (선택적)
- 사용자가 지정한 "가장 중요한 이벤트" 표시
- 설정되지 않았으면 비표시
- 3줄 레이아웃: 이벤트명 / 시간 / 태그명
- 클릭 → 이벤트 상세 페이지로 이동

#### 5.2.3 미완료 할일 목록 (Uncompleted)
- 기한이 지난 미완료 할일 목록
- 새로고침 버튼으로 수동 갱신 가능
- 각 항목:
  - 3줄 레이아웃: 할일명(빨간색) / 시간 / 태그명
  - 완료 버튼 (원형 체크박스)
- 반복 할일 완료 시: 범위 선택 다이얼로그 (이번만 / 이후 전체 / 모두)

#### 5.2.4 Current Todo 목록
- `is_current=true`인 활성 할일 목록
- 각 항목:
  - 3줄 레이아웃: 할일명 / 시간 / 태그명
  - 완료 버튼 (원형 체크박스)
- 완료 동작은 미완료 할일과 동일

#### 5.2.5 이벤트 목록 (Events)
- 선택된 날짜의 모든 이벤트 (할일 + 일정) 시간순 정렬
- 정렬 기준: `at` → timestamp, `period` → period_start, 시간 없음 → 마지막
- 각 항목:
  - 3줄 레이아웃: 이벤트명 / 시간 / 태그명
  - 클릭 → 이벤트 상세 페이지로 이동
- 이벤트 없을 때: "이벤트 없음" 빈 상태 표시

**하단 고정 영역**:

#### 5.2.6 Quick Todo 입력
- 텍스트 입력 필드
- Enter 키로 즉시 할일 생성 (기본 태그, `is_current=true`)
- 생성 실패 시 에러 토스트 표시

#### 5.2.7 새 이벤트 만들기 버튼
- 강조 버튼 (진한 배경)
- 클릭 → 타입 선택 팝업 (Todo / Schedule)
- 선택 → 해당 생성 페이지 모달 오픈

### 5.3 태그 필터링
- 모든 목록에 태그 필터 적용 (숨긴 태그의 이벤트는 비표시)

## 6. 키보드 단축키

| 키 | 동작 | 조건 |
|----|------|------|
| `n` | 새 할일 생성 모달 오픈 | 입력 필드에 포커스가 없을 때 |
| `Escape` | 모달/오버레이 닫기 | 모달이 열려있을 때 |

## 7. 데이터 흐름

### 7.1 이벤트 로딩
- 캘린더가 표시하는 월 범위에 해당하는 할일/일정을 API에서 조회
- 동일 범위 재요청 시 캐시 사용 (중복 호출 방지)
- 이벤트는 날짜별로 그룹핑되어 `eventsByDate` Map에 저장
- 다일 이벤트는 걸쳐있는 모든 날짜에 중복 등록

### 7.2 공휴일 로딩
- 캘린더에 표시되는 연도의 공휴일을 API에서 조회
- 연도 경계 (1월/12월)인 경우 인접 연도도 함께 로드
- 미니캘린더와 중앙캘린더 각각 독립적으로 로드

### 7.3 태그 필터
- `hiddenTagIds` Set으로 관리
- `localStorage`에 영속화
- 캘린더 이벤트 표시, 우측패널 모든 목록에 동시 적용

## 8. 네비게이션 패턴

- 이벤트 생성/수정은 **모달 네비게이션** 방식
  - `navigate(path, { state: { background: location } })` 패턴
  - 현재 페이지가 배경에 유지되고 위에 모달이 오버레이
  - 모달 배경 클릭 또는 Escape → 이전 페이지로 복귀
- 라우트:
  - `/todos/new` — 새 할일 생성
  - `/todos/:id/edit` — 할일 수정
  - `/schedules/new` — 새 일정 생성
  - `/schedules/:id/edit` — 일정 수정
  - `/events/:id` — 이벤트 상세
  - `/tags` — 태그 관리
  - `/done` — 완료된 할일
  - `/settings` — 설정
